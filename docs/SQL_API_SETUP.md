# SQL Account API Setup

The existing website, Google Sheet, and Apps Script backend are LeVince's lightweight ERP. `SqlApi.gs` is the private connector from that ERP to SQL Account.

## 1. Prepare SQL Account

On the Windows computer running SQL Account:

1. Open the SQL Account logon screen.
2. Open Settings, right-click the company database, and choose `Configure API Service...`.
3. Install and start the service.
4. Complete public IP and TCP port 443 forwarding if SQL Account requests it.
5. Run `Test Connection` until it shows `OK`.

## 2. Create the API user

1. In SQL Account, open `Tools > Maintain User > Detail`.
2. Create a dedicated user named `APIUser`.
3. Select `More > API Secret Key > Generate API Secret Key`.
4. Save both the Access Key and Secret Key immediately. This user is only for API access.

## 3. Configure Apps Script

In the Apps Script project, open `Project Settings > Script Properties` and add:

| Property | Value |
| --- | --- |
| `SQL_API_ACCESS_KEY` | Access Key generated by SQL Account |
| `SQL_API_SECRET_KEY` | Secret Key generated by SQL Account |
| `SQL_API_HOST` | `https://api.sql.my` |
| `SQL_API_REGION` | `ap-southeast-5` |
| `SQL_API_SERVICE` | `sqlaccount` |

Do not put either key in GitHub, `Code.gs`, or frontend JavaScript.

## 4. Controlled first test

1. Add `SqlApi.gs` to the same Apps Script project as `Code.gs`.
2. Run `sqlConnectionStatus` from the Apps Script editor and approve permissions.
3. Confirm the result has `ok: true`.
4. Prepare one paid MYR test invoice that has not been uploaded to SQL.
5. Run `sqlSyncPaidInvoices` from the Apps Script editor.
6. Verify the customer and Sales Invoice inside SQL Account before enabling website upload controls.

Only MYR invoices are enabled in the first version. Foreign-currency invoices are intentionally blocked until an exchange-rate rule is confirmed.
